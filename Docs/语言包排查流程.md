# Ditto 语言包排查流程

后续排查/补齐多语言时，按本文执行。目标是**从机制根因修复**，禁止只改一两处硬编码或只改一个语言文件。

---

## 0. 权威源与产物路径

| 用途 | 路径 |
|------|------|
| **唯一语言源（改这里）** | `Debug/Language/*.xml` |
| 本地 Debug 运行加载 | `{Ditto.exe 目录}\language\` → 即 `Debug\Language\` |
| CI Release 产物 | 从 `Debug\Language\` 复制到 `Release64\Language\` |
| 安装包 / 便携包 | Inno / `BuildPortableZIP.bat` 从 `Debug\Language\` 取 |

**禁止**再维护仓库根目录 `Language/`（已废弃）。发现双份目录时，以 `Debug/Language` 为准并删除冗余。

运行时查找：`CGetSetOptions::GetPath(PATH_LANGUAGE)` → exe 同目录下的 `language\`。

## 1. 多语言机制（先理解再改）

### 1.1 两种挂载方式

| 机制 | 加载位置 | 应用方式 | 典型用途 |
|------|----------|----------|----------|
| **对话框段落** | `Ditto_*` 节点 → `LANGUAGE_ARRAY` | `UpdateWindowToLanguage`：按控件数字 ID `SetWindowText` | 按钮、静态文本、分组框 |
| **菜单段落** | 同上数组 | `UpdateMenuToLanguage`：按菜单命令 ID `ModifyMenu` | 右键菜单、托盘菜单 |
| **命名字符串** | 段落内 `ID="SomeName"`（`ATOI==0`）或 `Ditto_String_Table` | `GetString` / `GetDeleteClipDataString` / `GetQuickPasteKeyboardString` | MessageBox、列表列头、命令描述 |

### 1.2 关键约定

- XML `ID` **必须**与 `resource.h` 控件 ID / 菜单命令 ID / `ActionEnums` 枚举序值一致。
- `UpdateWindowToLanguage` / `UpdateMenuToLanguage`：**外文为空则跳过**（界面仍显示 RC 英文）。
- `Get*String`：命名 ID 写入 `m_csID`；数字 ID 写入 `m_nID`。空外文时应回退到代码默认串（见 `GetDeleteClipDataString`）。
- 新加 `ActionEnumValues` **只能追加在枚举末尾**，且必须同步追加 `Ditto_Options_Quick_Paste_Keyboard` 同号 `Item`（`ActionEnums.h` 已注明）。

### 1.3 常用段落 ↔ 代码入口

| XML 段落 | 应用入口 | 主要文件 |
|----------|----------|----------|
| `Ditto_Options_General` | `UpdateOptionGeneral` | `OptionsGeneral.cpp` |
| `Ditto_Options_CopyBuffers` | `UpdateOptionCopyBuffers` | `OptionsCopyBuffers.cpp` |
| `Ditto_Options_Quick_Paste_Keyboard` | `UpdateOptionQuickPasteKeyboard` + `GetQuickPasteKeyboardString(enumId)` | `QuickPasteKeyboard.cpp` / `ActionEnums.cpp` |
| `Ditto_DeleteClipData` | `UpdateDeleteClipData` + `UpdateDeleteClipDataMenu` + `GetDeleteClipDataString` | `DeleteClipData.cpp` |
| `Ditto_Adv_Options` | `UpdateAdvOptions` | `AdvGeneral.cpp`（对话框 chrome：标题/按钮） |
| `Ditto_Adv_Properties` | `GetAdvPropertyString(settingId, default)` | `AdvGeneral.cpp`（属性网格设置名；`ID`=`SETTING_*`） |
| `Ditto_Right_Click_Menu` | `UpdateRightClickMenu` | `QPasteWnd.cpp` 等 |
| `Ditto_String_Table` | `GetString("Id", default)` | 全局提示/警告（含 Adv 分组/描述/脚本标题） |

## 2. 标准排查流程（按序执行）

### Step A — 复现与定位界面

1. 记下：**页面/对话框名称**、控件文案（英文原文）、操作（按钮 / 右键 / MessageBox / 列表命令）。
2. 在 `CP_Main.rc` 搜英文原文 → 得到控件 ID 或菜单 ID。
3. 在 `resource.h` 确认宏数值。

### Step B — 判断属于哪一类缺口

对每个未翻译点，归入一类：

| 类型 | 特征 | 根因通常是 |
|------|------|------------|
| **T1 对话框控件** | OnInit 已调 `Update*`，仍英文 | XML 缺该数字 ID，或 ID 写错段落 |
| **T2 菜单** | `LoadMenu` 后仍英文 | 未调 `UpdateMenuToLanguage`，或 XML 缺菜单命令 ID |
| **T3 MessageBox / 运行时串** | 代码里 `_T("...")` 硬编码 | 未走 `GetString` / `GetXxxString`，且 XML 无命名键 |
| **T4 枚举/命令列表** | 选项里一长串命令名 | `GetQuickPasteKeyboardString(枚举值)` 缺 XML Item |
| **T5 双份/过期目录** | Debug 与根目录不一致 | 未以 `Debug/Language` 为唯一源 |

**禁止**：只改 RC 英文；只改简体一个文件却声称“全量”；只在代码里写中文默认值（无 `/utf-8` 时 CI 会乱码）。

### Step C — 对照权威语言文件

以 `Debug/Language/English.xml` 为 schema，以 `Chinese Simplified.xml` 为中文验收：

1. 打开对应 `Ditto_*` 段落。
2. 确认是否存在 `ID="<resource.h 数值>"` 或命名 ID。
3. 确认外文非空（中文包必须有中文；英文包对需 `Get*String` 的键应填英文外文，避免返回空串）。
4. 用脚本扫 **全部** `Debug/Language/*.xml`，统计缺 ID 的文件列表（不允许“只修中英、跳过其余”）。

### Step D — 对照代码接线

1. 对话框：`OnInitDialog` 是否调用对应 `Update*`。
2. 菜单：`LoadMenu` / `GetSubMenu` 之后是否立即 `Update*Menu`。
3. MessageBox：是否 `GetString` / `GetDeleteClipDataString` 等，禁止裸英文。
4. 命令列表：是否 `ActionEnums::EnumDescription` → `GetQuickPasteKeyboardString`。

缺接线则先补 API（保持与现有 `UpdateRightClickMenu` 等同一模式），再补 XML。

### Step E — 修复顺序（根因优先）

1. **先定 schema**：在 `English.xml` 补齐 `Item`（`English_Text` 与代码/RC 默认一致）。
2. **再定中文**：`Chinese Simplified.xml`（及繁体若需要）写完整译文。
3. **批量同步结构到全部语言包**：其它语言至少具备相同 ID；译文优先复用已有同义项，否则可先用英文外文占位（保证 `Update*` 能命中），但不得缺 ID。
4. **改代码接线**（若属 T2/T3）：最小改动，复用现有 `CMultiLanguage` 方法，不新增平行系统。
5. **清理错误 ID**：如控件已从某页移除，删除其它段落里的孤儿 ID，避免误导。

### Step F — 验收清单

对每个修复点勾选：

- [ ] `resource.h` / 枚举 ID 与 XML `ID` 一致  
- [ ] 对应 `Update*` / `Get*` 已接线  
- [ ] `English.xml` schema 完整  
- [ ] `Chinese Simplified.xml` 关键可读  
- [ ] **全部** `Debug/Language/*.xml` 含新 ID（脚本审计 0 missing）  
- [ ] 无根目录 `Language/` 残留  
- [ ] CI/Inno 仍指向 `Debug\Language`  
- [ ] 未把中文写进 `.cpp` 默认字符串  

### Step G — 提交与构建

- 语言与接线变更一并提交；**不要**只提交部分语言文件。  
- 本地无 VS 时仅通过 GitHub Actions（`feature/**` push）验证；产物中的 `Language\` 应来自 `Debug\Language`。

## 3. 高频场景速查

### 3.1 选项 → 快速粘贴按键「部分命令」英文

1. 打开 `ActionEnums.h`，确认枚举序值（从 `FIRST_ACTION=0` 递增）。  
2. 打开 `ActionEnums.cpp` 的 `EnumDescription`，记下英文默认串。  
3. 检查 `Ditto_Options_Quick_Paste_Keyboard` 是否有 `ID="<序值>"`。  
4. 缺失则全语言包补齐；`English_Text` 必须与 `EnumDescription` 一致（按 ID 查找，不按英文匹配）。

### 3.2 删除剪贴数据：确认框 / 右键菜单

| UI | 正确做法 |
|----|----------|
| 删除确认 | `GetDeleteClipDataString("DeleteSelectedConfirm", L"...")` + XML 命名项 |
| 右键 | `LoadMenu` → `UpdateDeleteClipDataMenu` + XML 数字项 `32980/32981/32982` |
| 对话框按钮/标签 | `UpdateDeleteClipData` + 控件 ID（如 Close=`8`/`IDCLOSE`，Compact=`1028`） |

注意：主列表右键 `Ditto_Right_Click_Menu` 的 ID 与删除页菜单 ID **不同**，不能复用。

### 3.3 选项通用：Preview / 复制缓冲区 4–5

- Preview：`IDC_BUTTON_PREVIEW_THEME` = **2175** → `Ditto_Options_General`  
- 复制缓冲区 4/5：以 `resource.h` 的 `IDC_BUFFER_GROUP_4/5` 等为准；Win Cut 正确 ID 为 **2114** / **2124**（勿用错误的 2119/2129）  
- 勿把已挪走的 Compact 按钮 ID 留在 `Ditto_Options_General`

### 3.4 压缩修复警告

- 使用 `GetString("CompactRepairWarning", ...)`（`Ditto_String_Table`）  
- 改英文默认句时，同步更新所有语言包中该 ID 的 `English_Text`（及中文外文）

### 3.5 选项 → 常规 → 高级（属性网格）

**根因约束**：`SETTING_*` 从 1 起，与 `IDOK`/`IDCANCEL` 撞号，**禁止**把属性名放进 `Ditto_Adv_Options`（否则 `UpdateAdvOptions` 会改掉确定/取消）。

| UI | 正确做法 |
|----|----------|
| 对话框按钮/标题 | `UpdateAdvOptions` + `Ditto_Adv_Options`（含 Next `2124`=`>`） |
| 属性网格设置名 | `GetAdvPropertyString(SETTING_*, L"英文默认")` + `Ditto_Adv_Properties` 数字 ID `1`…`111` |
| 分组/描述/脚本标题 | `GetString("AdvGroup_*" / "AdvDesc_*" / "Adv*Title", …)` → `Ditto_String_Table` |
| True/False 取值 | **保持英文**（存盘比较 `L"True"`）；只译属性显示名 |

`LoadSection` 只加载**有外文子节点**的 `Item`；英文包也必须写外文正文（可与 `English_Text` 相同）。  
新增设置：追加新的 `SETTING_*`（勿复用已删 ID）→ `AdvGeneral` 用 `AdvProp`/`GetAdvPropertyString` → 全语言包补同号 `Item`。  
可参考 `scripts/sync_adv_properties_i18n.py` 做结构同步与审计。

## 4. 推荐审计脚本逻辑（可临时编写）

对 `Debug/Language/*.xml` 批量检查：

```
对每个文件:
  必需段落存在
  必需 ID 集合每个恰好出现 1 次
  Options General 无孤儿 Compact ID 1028（若控件已不在该页）
  CopyBuffers 无错误 Win ID 2119/2129
  QuickPasteKeyboard 含全部 UserConfigurable 枚举 ID（当前至 SHOW_STARRED_CLIPS=140）
  DeleteClipData 含 Close=8、Compact=1028、DeleteSelectedConfirm、32980-32982
  Adv_Options 含 2124；Adv_Properties 含 SETTING 1..111 且每项有外文
  String_Table 含 AdvGroup_Ditto / AdvGroup_RegexExclude / AdvDesc_* / Adv*ScriptsTitle / AdvResetClipOrderTitle
  Chinese Simplified 抽查关键中文非空且非英文原样
```

退出码：有缺失则非 0。修复后必须再跑一遍至通过。

```
5) 用脚本审计全部 xml 至 0 missing 后再汇报
具体问题：<在此描述页面与未翻译文案>
```
